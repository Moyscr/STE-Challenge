# Bug Report

## Bug ID
BUG-004

## Description
When creating a PUT request for an employee with an invalid ID, the API returns a 405 error code when it should be 404

---

## Test Environment

| Field          | Details            |
|----------------|--------------------|
| Browser        |        Chrome            |
| Browser Version|         147.0.7727.15           |
| Date Reported  |         03/05/2026          |

---

## Steps to Reproduce

1. Send a PUT request for a non-existent employee, something like 'id: 00000_0000_00000'
2. Look at the message returned by the API

---

## Actual Result

API returns 405 METHOD NOT ALLOWED

---

## Expected Result

API should return 404 NOT FOUND
