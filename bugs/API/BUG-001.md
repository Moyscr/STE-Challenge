# Bug Report

## Bug ID
BUG-001

## Description
When submitting a POST or PUT request with a username different from 'TestUser959', the value does not persist, always defaulting to 'TestUser959' with no message or response of why this is being changed.

This may be confusing to the user, also leaving the id as the only unique field for employees

## Steps to Reproduce

1. Make either a POST request to create a new employee with username 'test123'
2. Fetch the just created user using a GET request with the employee id
3. Look at the username field

---

## Actual Result

username defaults to 'TestUser959'

---

## Expected Result

Either the userame set by the request has to persist, or there should be a clear message explaining why this field is automatically modified
