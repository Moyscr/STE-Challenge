# Bug Report

## Bug ID
BUG-008

## Description
Login with wrong credentials returns 405 error instead of UI feedback as an error message for the user

---

## Test Environment

| Field          | Details            |
|----------------|--------------------|
| Browser        |        Chrome            |
| Browser Version|         147.0.7727.15           |
| Date Reported  |         03/05/2026          |

---

## Steps to Reproduce

1. Go to the login page of the system
2. Fill the log in fields with wrong credentials e.g. Username: user, Password: pass
3. Click the 'Log In' button

---

## Actual Result

Submitting the login form with invalid credentials navigates the user to an unhandled 405 error page instead of displaying inline validation feedback.

**Screenshot:**

![Actual Result](/bugs/screenshots/bug_008_actual.png)

---

## Expected Result

The system should handle the invalid credential providing UI feedback to the user as an error message
